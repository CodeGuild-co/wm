Let's Tool Up

Playing with new tools is one of the most fun things to do as a software engineer

We spoke yesterday about problem solving by being lazy, finding a tool that does a task you thought you were going to have to do is one of the best lazy things you can do

There are some very standard types of tools that we're going to cover in this session. These are the bread and butter tools that you'll use everyday and for hours and hours

There are also very task specific tools that we won't look at. You'll just have to remember to be lazy the next time you start a task. Ask yourself "is there a tool out there that would do this for me?"

Operating Systems

Just a quick aside...

The main operating system types you'll care about as a programmer are Windows and Unix

Unix is the basis of the Linux operating system (which Debian is a flavour of) and Apple's OS X. It's open source and very programmer friendly. You'll almost certainly need to know how to use some form of Unix

Windows is its own beast, you probably won't need to know Windows as a software engineer but it won't hurt you

Your operating system is a tool that you pick, just like any other tool. You'll want to think about which one you use and I would recommend using lots of them to see which ones you like

This week we'll be using Ubuntu, it's a very popular choice. Try others (especially inside VirtualMachines where thigns can't go wrong)

Editing Text

Programming is writing instructions to a computer in a language it understands

As a software engineer you'll spend a lot of time typing text in a very specific way

Computers are not very forgiving if you make a spelling or grammar mistake

For these reasons, the tool you pick to write your software is very important

Software engineers have made and lost life-long friends over choices in text editor
